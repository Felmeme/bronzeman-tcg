# PvM Content -> Monster Card Rosters (Bronzeman TCG)

Resolution: in-game NPC name (lowercased) -> `tracked_monster_names.json` entityToCards -> exact-matched against `Card.json`. Generated by `build_content_cards.py`.

## KEY ARCHITECTURAL CAVEAT (read first)

The task framed unresolvable NPCs as making content "uncompletable". Per this plugin's own design (CLAUDE.md: *"Entities not in a snapshot are never restricted"*), an NPC with **no monster card is never blocked** - it is freely attackable. So a missing card does **NOT** make content uncompletable; it only means that NPC cannot appear in the require-all card list. Every content below is completable by a strict player. The one thing to surface in the panel: the require-all list for **Chambers of Xeric** is drastically smaller than its real roster because almost none of its bosses are card-backed.

## Judgment calls

- **Fortis / Passionate Supporter:** has a card but is an arena spectator you do not need to fight; excluded from the require-all list so the panel doesn't demand an irrelevant card. Bee Swarm and Healing totem are genuine attackable wave objects with no card.

- **CoX / Guardian:** in-game NPC name is "Guardian"; the snapshot's "Stone guardian" (-> `Stone guardian` card) is a different entity and does not resolve by exact name.

- **CoX optionality:** rooms are randomised; only Great Olm is guaranteed, so the practical minimum can be just `Great Olm`. Lizardman shaman only if the Shamans room rolls.

- **ToA path choice:** the four Wardens are always fought; the four bosses and their path adds only appear on paths the player's Invocations unlock, so the true minimum is a subset of the listed union.

- **ToB / Nylocas Prinkipas:** Hard-Mode-only miniboss; still card-backed so kept in the union. Verzik's phases fold to one card.

- **Yt-HurKot** deliberately appears in both Fight Caves and Inferno rosters (Jad's healers in each); deduped within each content.
